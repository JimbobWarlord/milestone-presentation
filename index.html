ASSIGNMENT 2: MILESTONE PRESENTATION

CONCEPTUAL RESPONSE

My project is a ‘music map’ of the hit indie game, ‘Hollow Knight’, which serves as both a fun, interactive experience and an homage to the newly released ‘Hollow Knight: Silksong’. This is in response to the ‘Spatial Control’ prompt.

Within the experience, the user can hover of the areas of the map to play various themes from the game; for example, if the user hovers over an area called ‘Greenpath’, the song ‘Greenpath’ will start playing. The page also comes with several additional features, ranging from a convenient music list with all tracks from the game, an info page so users can easily understand the experience, a random feature that I’ll explain in detail in the next section, and adjustable settings that can alter the music.

The overall design and layout are heavily based on ‘Hollow Knight’ itself, featuring a primary similar font (Trajan Pro-Bold), and incredibly similar visuals. The additional pages such as the music list and settings screens are inspired by the menus of the game.

TECHNICAL APPROACH

Before working on the prototype, I created a mockup of the website page, done in Photoshop. When making the mockup, I decided to use the detailed Hollow Knight map given to Kickstarter backers before the game’s release, which conveniently has all of the key locations.

My chosen JavaScript technique was the random feature (Math.random()). By pressing a button, the user’s cursor location would move to a random position, thereby playing a random song. Originally, the function was simply going to be playing a random song from the music list, but that felt like it would be too simple.

PEER TESTING

Prompt 1: As a way to spice up my project, I'm looking to incorporate settings that can adjust aspects of the soundtrack, such as a song's speed, pitch, volume and potentially added reverb. What other settings should be implemented to spice things up/make the soundtrack sound different?

A lot of testers thought the initial ideas of adjusting speed, pitch and volume were fine, though a few suggested implementing options to add ambience such as sound effects within ‘Hollow Knight’ that could make it more atmospheric. I really like this idea and I’m all for applying it to the project.

Prompt 2: A key part of Hollow Knight is the plethora of bosses within the game, though the map doesn't display where the boss fights take place. If you have a steel-trap memory, you could probably remember where the fights take place, but most viewers wouldn't know the specific locations. In my opinion, some of the best tracks are found within the boss fights, so I would like to implement these music pieces within the map in addition to the music list. Any ideas on how this could work?

There were many suggestions for how bosses could be implemented within the map; some of my favourite suggestions were the addition of pins over the map which signify where the bosses are, or potentially adding a button that toggles a separate screen which shows the boss locations. However, my favourite suggestion was adding images from the game’s ‘Hunter’s Journal’ which work as indicators, and though that would probably require the most work, it’d definitely be rewarding. I’m pleasantly surprised by the number of testers who have played/are familiar with the game!

THE ISSUE

So you’ve probably noticed by now, and this is something I’ve been neglecting to talk about so I could dedicate this section to it, copyright is a big issue. Due to the project being public, it’s very likely I won’t be able to implement anything from the game ranging from the map itself to all of Christopher Larkin’s music. Therefore, my workaround is to attempt to find suitable replacements for the music as well as tracing over the map myself to devoid it of copyright.
